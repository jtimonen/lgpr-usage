readLiuData <- function(dataDir, protein){
  fn_X   <- paste(dataDir,"/liu_preproc_X.csv",sep="")
  fn_Y   <- paste(dataDir,"/liu_preproc_Y.csv",sep="")
  X_data <- read.csv(fn_X, header=TRUE, sep=",")
  X_data <- X_data[,1:5]
  Y_data <- read.csv(fn_Y, header=TRUE, sep=",")
  names  <- colnames(Y_data)
  if(!is.character(protein)){
    pname <- names[protein]
  }else{
    pname <- protein
  }
  
  cat("Read data for protein '", pname, "'. \n", sep = "")
  y      <- Y_data[[pname]]
  notnan <- which(!is.nan(y))
  n_nan  <- length(which(is.nan(y)))
  data   <- data.frame(cbind(X_data, y))
  data   <- data[notnan, ]
  cat("Removed ", n_nan , " rows with NaN value for the response variable.\n", sep = "")
  return(data)
}

4. Component selection methods¶

To further assess the component relevances for the heterogeneous model, we first we plot the distribution of component relevances over all posterior samples.

In [10]:
options(repr.plot.width=12, repr.plot.height=8) 
plot(fit2) + theme(text=element_text(size=FONT_SIZE))

We can plot the selection probabilities as a function of the threshold.

In [11]:
pfun <- function(x){stats::dbeta(x, 100, 5)}
sel <- selection_prob(fit2, p = pfun, h = 0.001)
options(repr.plot.width=11, repr.plot.height=6)
sel$plot + theme(text=element_text(size=FONT_SIZE))

To avoid having to fix a specific threshold value, we can give a distribution $p(\tau)$ to weigh different threshold values. The selection probability for component $j$ is then obtained as $$ p_j = \int_0^1 p(\tau)p_j(\tau) d\tau, $$ where $p_j(\tau)$ is the selection probability of component $j$ using fixed threshold $\tau$. Here we used the $Beta(100,5)$ distribution to weight thresholds. This distribution and the weighted selection probabilities are shown below.
